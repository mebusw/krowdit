[edit] Unit test with Test Coverage

  • SenTest integrated in XCode
  • see Coverstory GUI or http://www.milesdennis.com/2011/04/code-coverage-in-xcode-4.html for test coverage
  • change a few settings in the build to link in the gcov library, and turn on code coverage. In the build settings, under Linking, add the -lgcov to "Other Linker Flags". Then under the Code Generation section change "Generate test coverage file" and "Instrument Program Flow" to Yes.
  • change the compiler version to GCC 4.2, not LLVM GCC 4.2.
  • locate the output files (CodeCover is the project name). These are located deep in the Build folder (which in XCode 4 by default will be something like ~/Library/Developer/XCode/DerivedData/CodeCover.../Build)

and under that will be Intermediates/CoverCode.build/Debug-iphonesimulator/CodeCover.build and finally you get to the Objects-normal folder. If you look in here you should see a bunch of gcda and gcno files. http://coverstory.googlecode.com/svn/site/images/settings.png
